consul角色介绍

一、启动agent

1.若要使用consul，必须运行agent。agent可以运行为server或client模式，每个数据中心必须至少拥有一台server，建议在一个集群中有3或者5个server.一方面可以防止在出现失败时造成数据丢失；另一方面随着添加更多的机器，一致性会逐渐变慢。这在故障情况下的可用性和性能之间取得了平衡。但是，客户端的数量没有限制，可以很容易地扩展到数千或数万。

2.运行agent server

开发模式启动单节点：
consul agent -dev -ui -client 0.0.0.0



运行集群：四个server，一个client

hallen-集群IP:consul agent -server -bootstrap-expect 5 -data-dir /tmp/consul -node=s_hallen -bind=192.168.0.106 -ui -rejoin -client 0.0.0.0 -join 192.168.0.106

hallen1-服务端:consul agent -server -bootstrap-expect 5 -data-dir /tmp/consul -node=s_hallen1 -bind=192.168.0.109 -ui -rejoin -client 0.0.0.0 -join 192.168.0.106
hallen2-服务端:consul agent -server -bootstrap-expect 5 -data-dir /tmp/consul -node=s_hallen2 -bind=192.168.0.111 -ui -rejoin -client 0.0.0.0 -join 192.168.0.106
hallen3-服务端:consul agent -server -bootstrap-expect 5 -data-dir /tmp/consul -node=s_hallen3 -bind=192.168.0.112 -ui -rejoin -client 0.0.0.0 -join 192.168.0.106

hallen4-客户端:consul agent -data-dir /tmp/consul -node=c_hallen4 -bind=192.168.0.113 -ui -rejoin -join 192.168.0.106




先运行集群IP的agent，然后运行其他的server或者client，使用-join加入进去，确保加入之前存在该节点





-server ： 定义agent运行在server模式
-bootstrap-expect ：在一个datacenter中期望提供的server节点数目，当该值提供的时候，consul一直等到达到指定sever数目的时候才会引导整个集群，该标记不能和bootstrap共用
-data-dir=/tmp/consul 目录
-node=s1 节点在集群中的名称，在一个集群中必须是唯一的，默认是该节点的主机名
-bind：该服务器节点名,该地址用来在集群内部的通讯，集群内的所有节点到地址都必须是可达的，默认是0.0.0.0
-ui-dir(-ui)： 提供存放web ui资源的路径，该目录必须是可读的
-rejoin：使consul忽略先前的离开，在再次启动后仍旧尝试加入集群中。
-config-dir：配置文件目录，里面所有以.json结尾的文件都会被加载
-client：consul服务侦听地址，这个地址提供HTTP、DNS、RPC等服务，默认是127.0.0.1所以不对外提供服务，如果你要对外提供服务改成0.0.0.0

ui界面：http://127.0.0.1:8500/

3.运行agent client

consul agent -data-dir /tmp/consul -node=c1 -bind=192.168.0.113 -config-dir=/etc/consul.d/ -join 192.168.1.104


-join 加入到已有的集群中去，集群地址

二、查看集群成员

consul members

三、停止agent

consul leave
