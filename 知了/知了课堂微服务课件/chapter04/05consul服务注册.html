consul服务注册

一、配置文件

1.注意事项

  • 以.json结尾的文件

  • 配置信息json格式

2.配置信息

{
  "service":{
    "id": "consul-service-demo",
    "name": "consul-service-demo",
    "address": "127.0.0.1",
    "port": 8080,
    "tags": ["uth","report"],
    "checks":[
      {
        "http": "http://127.0.01:8080/health",
        "interval": "10s"
      }
    ]
  }
}

3.配置参数详解

id：节点id

name:节点name

address:节点IP

port：节点端口

tags：节点标签，是个列表

checks：服务检查，列表
    1.Script+ Interval
        {
            "id": "mem-util",
            "name": "Memoryutilization",
            "script": "/usr/local/bin/check_mem",    // 必须是shell脚本
            "interval": "10s",
            "timeout": "15s"    // 默认是30s
          }
    2.HTTP+ Interval
        请求超时时间等于调用请求的间隔时间
        {
            "id": "api",
            "name": "HTTPAPI on port 5000",
            "http": "http://localhost:5000/health",
            "interval": "10s",
            "timeout": "15s"
        }

    3.TCP+ Interval
        {
          "check": {
            "id": "ssh",
            "name": "SSHTCP on port 22",
            "tcp": "localhost:22",
            "interval": "10s",
            "timeout": "1s"
          }
        }



datacenter：同命令行参数-datacenter

data_dir：同命令行参数-data_dir

bootstrap：同命令行参数-bootstrap

bootstrap_expect：同命令行参数-bootstrap_expect

bind_addr：同命令行参数-bind

enable_syslog：同命令行参数-syslog

log_level：同命令行参数-log_level

node_name：同命令行参数node

server：是否是server节点

4.加载配置文件

-config-dir=D:\go\consul_conf

1.Server节点保存数据

2.Client负责健康检查及转发数据请求到Server
